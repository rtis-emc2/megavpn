# Security and Release Review: 7.1.1.11

**Release:** `7.1.1.11`

## Decision

This release isolates privileged package maintenance from the long-running
agent without weakening the agent service sandbox globally. The full audit
found no unresolved critical or high-severity issue in the reviewed control
plane, agent, job, secret, filesystem and VPN configuration paths. It is
suitable for production promotion after the documented release gates pass and
the operator completes a staged node smoke test.

## Audit Coverage

- Operator authentication, session cookies, CSRF enforcement, password
  hashing, RBAC and public share-token lifecycle.
- Signed agent requests, replay protection, enrollment and token rotation,
  job lease ownership and forged-result rejection.
- PostgreSQL query construction, secret-reference hydration, persisted job
  redaction and audit events.
- Static web assets, binary artifacts, temporary files and symlink/path
  containment.
- Privileged agent commands, package repair/install/purge, runtime takeover,
  emergency cleanup and full-node wipe.
- Nginx, OpenVPN, WireGuard, Xray/VLESS, Shadowsocks and L2TP/IPsec input
  validation, configuration rendering and cleanup.

## Findings Closed

- Full-node wipe now fails closed when `dpkg-query` cannot establish package
  state or unmanaged L2TP/IPsec executables remain. Agent self-removal is not
  scheduled after an incomplete purge.
- The transient root package helper now uses a strict apt/dpkg action
  allowlist, exact apt option and environment-variable allowlists, safe package
  argument validation and a validated systemd unit name.
- HTTP rate-limit and signed-request replay maps have hard capacity bounds to
  prevent unauthenticated memory growth. New rate-limit identities fail closed
  at capacity instead of evicting an existing protection window.
- Static web asset resolution verifies the real path after symlink expansion
  and rejects traversal outside the configured web root.
- Malformed operator invite resend bodies now fail with HTTP 400 instead of
  silently falling back to default values.
- VLESS client-access-group materialization no longer hides metadata update
  failures that could leave a partially synchronized service access.

## Security Properties

- The long-running agent retains `NoNewPrivileges`, `RestrictSUIDSGID` and its
  existing filesystem restrictions.
- Transient delegation accepts only direct or `env`-wrapped `apt-get` and
  `dpkg` commands generated by fixed runtime installers.
- Provider-controlled endpoint, username, password, key and uploaded config
  values are never accepted as executable names or package names.
- The helper receives a unique unit name, a nine-minute runtime limit and the
  fixed system executable path.
- The outer install operation retains its ten-minute context deadline.
- The helper is collected after completion and its name, exit code and bounded
  combined output are retained in job evidence.
- Existing external egress runtime is not replaced until package installation
  and executable verification succeed.
- L2TP takeover removes only UUID-shaped MegaVPN deployment paths and exact
  MegaVPN IPsec include lines; it does not use provider-controlled filesystem
  paths.
- Package purge is limited to a fixed L2TP/IPsec package allowlist and runs
  only during explicitly confirmed `full_node` wipe.
- A failed full-wipe package purge prevents delayed agent self-removal, keeping
  a recovery channel available.

## Failure Model

If transient-unit creation is unavailable, existing non-systemd/manual agent
execution keeps the direct package path. A repository, lock, dependency or
maintainer-script failure terminates apply and reports the final meaningful
diagnostic lines. It does not mutate provider credentials or activate a
partially configured deployment.

## Residual Risk

Operating-system package maintainer scripts execute as root, as required by
normal package installation. Node administrators must use trusted,
authenticated Ubuntu repositories and monitor package/repository drift. A
broken node package database may still require the documented manual recovery
sequence. L2TP apply deliberately stops the distribution XL2TPD service because
the managed external gateway owns UDP/1701 on that node.

External gateway services require root and network-administration privileges
for tunnel and policy-routing setup. Further systemd capability reduction is
deferred until protocol-by-protocol staged data-plane tests prove the minimum
capability set. Proxy-derived client addresses are trustworthy only when the
control plane is deployed behind a configured reverse proxy that overwrites
forwarded headers. These are documented deployment constraints rather than
open application vulnerabilities.

## Verification

Regression tests verify package-helper allowlisting, incomplete purge
rejection, unmanaged executable detection, bounded unauthenticated caches,
request replay rejection, asset traversal and symlink rejection, job lease
ownership, public token revocation/expiry, SSH host-key changes, Nginx
directive/header injection, VPN configuration injection and VLESS group
consistency. Full release-gate output provides the final build, race, vet,
JavaScript, shell, documentation and vulnerability-scan evidence.
