# Security and Release Review: 7.1.0.3

**Release:** `7.1.0.3`

## Scope

This review covers the first runtime traffic-accounting collector added after
`7.1.0.2`:

- managed Xray/VLESS configs can render a loopback-only Xray Stats API;
- default Xray service-pack components enable traffic accounting in their specs;
- migration `000014_xray_traffic_accounting_defaults` updates existing default
  service-pack templates;
- `megavpn-agent` collects Xray user uplink/downlink counters and submits
  aggregate buckets through the signed agent endpoint;
- PostgreSQL ingest can attribute Xray samples by client username and service
  access when the agent only knows the Xray user/email;
- regression tests cover config rendering, parsing and delta aggregation.

## Security Posture

The collector stores aggregate byte counters only. It does not capture packet
payloads, URLs, headers, bodies, DNS queries or TLS session contents.

The Xray Stats API generated by the Control Plane is bound to `127.0.0.1` and is
not exposed through the public VLESS/Nginx endpoint. The agent refuses to use an
Xray stats inbound if the config shows a non-loopback listen address.

Agent submission continues to use node bearer-token authentication and signed
messages. Server-side ingest validates node ownership for instance references,
rejects malformed UUIDs and negative/empty counters, and derives client/service
access attribution inside the Control Plane instead of expanding agent database
responsibility.

## Residual Risk

- Existing Xray instances must be re-applied so their rendered config contains
  the loopback Stats API.
- The collector keeps in-memory absolute-counter baselines. After agent restart,
  the first interval is baseline-only and does not submit historical cumulative
  counters.
- OpenVPN and WireGuard collectors are still pending and must be implemented
  with protocol-specific attribution.
- High-volume deployments may still need PostgreSQL partitioning after real
  traffic cardinality measurements.

## Verification

- `go test ./...`
- `scripts/docs-consistency.sh`
- `scripts/release-gate.sh`

PostgreSQL integration with `MEGAVPN_TEST_DATABASE_DSN` remains recommended for
upgrade proof on disposable databases.
