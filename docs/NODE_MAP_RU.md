# Карта нод

**Релиз:** `7.0.1.25`

Карта нод - это рабочая область топологии для примерного расположения нод и
визуализации управляемых backhaul-связей. Оператор не вводит широту, долготу или
метку дата-центра вручную. Control Plane определяет данные по публичному адресу
ноды через настроенный GeoIP endpoint и хранит результат как кеш в записи ноды.

English companion: [NODE_MAP.md](NODE_MAP.md).

## Модель данных

Карта использует следующие данные:

| Данные | Источник | Примечания |
| --- | --- | --- |
| Маркер ноды | `nodes.address` плюс кеш GeoIP | Разрешаются только публичные IP. Private, loopback, multicast и link-local адреса пропускаются. |
| Страна, регион, город | Ответ GeoIP | Данные примерные; это ориентир для эксплуатации, а не доказательство для биллинга или комплаенса. |
| Владелец сети | GeoIP `org`, `isp`, `asn`, `company`, `traits` или эквивалентные поля | Обычно показывает хостинг-провайдера, оператора связи, облако или владельца сети дата-центра. |
| Источник GeoIP | Hostname провайдера и resolved IP | Показывается в UI, чтобы оператор понимал источник данных. |
| Backhaul-связь | `backhaul_links` и выбранный transport | Рисуется только когда обе endpoint-ноды имеют координаты. |

## Поток выполнения

1. Оператор создает или редактирует ноду и указывает ее публичный адрес.
2. API извлекает host/IP из адреса.
3. Если адрес private/internal, GeoIP пропускается и внешний provider не
   вызывается.
4. Если адрес публичный, API запрашивает настроенный GeoIP endpoint.
5. Страна, город, координаты, ASN и владелец сети кешируются в таблице `nodes`.
6. Страница Node Map рисует локальную статичную карту мира, pin-маркеры нод,
   панель выбранной ноды и backhaul-связи, которые можно корректно отобразить.

Запросы списка нод также обновляют устаревший или отсутствующий GeoIP cache
небольшими пачками, которые ограничены `MEGAVPN_GEOIP_AUTO_ENRICH_LIMIT`.

## Конфигурация

Runtime-переменные в `/etc/megavpn/megavpn.env`:

| Переменная | Значение по умолчанию | Назначение |
| --- | --- | --- |
| `MEGAVPN_GEOIP_LOOKUP_URL_TEMPLATE` | `https://ipapi.co/{ip}/json/` | HTTPS endpoint template для Control Plane. Значение `disabled` отключает automatic lookup. |
| `MEGAVPN_GEOIP_TIMEOUT` | `3s` | Timeout одного lookup request. |
| `MEGAVPN_GEOIP_AUTO_ENRICH_LIMIT` | `5` | Максимум нод, которые обновляются за один node list request. |

В регулируемых средах используйте внутренний GeoIP service. Внешний GeoIP
provider будет видеть публичные IP нод, по которым выполняется lookup.

## Поведение UI

Страница сделана map-first:

- карта использует встроенный статичный asset карты мира и не требует ручного
  ввода координат или загрузки внешних изображений карты;
- клик по pin-маркеру выбирает ноду и открывает inspector на этой же странице;
- inspector показывает адрес, роль, статус, город/страну, координаты, владельца
  сети, ASN/source metadata и связанные backhaul;
- ноды без координат показываются в компактном блоке pending GeoIP;
- компактный список mapped nodes служит selector, а не заменяет страницу
  управления нодами.

## Отрисовка Backhaul

Карта накладывает managed ingress-to-egress backhaul как направленные линии
между pin-маркерами:

- active/applied transports отображаются как здоровые связи;
- provisioning/degraded links отображаются как warning-связи;
- failed links отображаются как failed-связи;
- disabled links остаются видимыми как выключенная topology и исключаются из
  managed route-policy selection до повторного включения;
- deleted links не показываются;
- links с unresolved координатами учитываются в summary, но не рисуются;
- если route metric настроена, она показывается на линии;
- под картой отображаются компактные topology cards с направлением, driver,
  metric, endpoint и статусом выбранного транспорта;
- route switch в каждой topology card ставит managed enable/disable jobs для
  выбранной связи и показывает типы jobs, которые API принял в очередь;
- topology cards показывают активные backhaul cleanup/apply jobs и jobs
  route-policy на ingress-ноде, пока convergence еще не завершен;
- ошибки выбранного transport показываются прямо на topology card, чтобы
  оператор видел первичную причину failed-связи без перехода со страницы карты.

Страница Backhaul остается источником правды для apply, probe, cleanup и
детальной диагностики transport. Node Map - это визуальная топология, а не
замена операционной диагностике.

## Правила безопасности

- Resolver принимает только HTTPS GeoIP templates с `{ip}` placeholder.
- Resolver отклоняет private, loopback, link-local, multicast и unspecified
  addresses до любого provider call.
- Response body ограничен по размеру и парсится как JSON.
- GeoIP best-effort: ошибки lookup не блокируют создание или обновление ноды.
- Фон карты отдается из локальных web assets; CSP не требует внешнего image
  source для topology view.

## Эксплуатационные примечания

- Координаты приблизительные и могут указывать на provider POP, город, регион
  или зарегистрированную локацию сети.
- Владелец сети может показать оператора связи или хостинг-провайдера, но не
  обязательно точный физический дата-центр.
- После изменения GeoIP settings перезапустите API service.
- Existing nodes дообогащаются автоматически после migration, когда UI или API
  запрашивает список нод.
