# Release 7.1.1.18

**Release:** `7.1.1.18`

Release `7.1.1.18` stabilizes L2TP over IPsec external-provider deployment,
probe and cleanup after partial or failed runtime installation.

## Scope

- Stops stale managed L2TP services and releases their PPP interface before a
  new Apply starts.
- Makes Cleanup ownership-aware and ordered: stop the managed runtime, remove
  only an ownership-verified UDP/1701 listener, release the PPP interface, then
  delete managed artifacts.
- Preserves runtime artifacts when Cleanup cannot prove that teardown is safe,
  so a later retry still has the ownership evidence it needs.
- Waits for both the L2TP interface and its IPv4 address before installing the
  provider default route.
- Requires the systemd unit, PPP interface, interface address, provider route,
  policy rule and an established and installed IPsec SA before reporting an
  L2TP deployment healthy.
- Requires two consecutive complete L2TP probes to avoid promoting a
  short-lived or partially converged runtime.
- Reports the exact missing runtime layers instead of the generic
  `external egress runtime is incomplete` result.

## Operator Action

Deploy `7.1.1.18` to the Control Plane and update the affected provider-node
agent to `7.1.1.18`. Retry `Cleanup` for the failed deployment, confirm that no
unmanaged process owns UDP/1701, then run `Apply`.

Do not terminate an unrelated system L2TP service merely to make Apply pass.
The platform only removes a UDP/1701 listener when it can associate the process
with the managed deployment.

## Compatibility

- No PostgreSQL migration is required.
- API and external-egress job payload contracts are unchanged.
- Existing non-L2TP external-egress profiles keep their previous probe
  requirements.
- Existing active L2TP profiles are revalidated with the stronger probe after
  the agent update.

## Validation

Agent tests cover stale-interface release, ordered cleanup, ownership-safe
listener handling, PPP options, route convergence, complete and incomplete
L2TP probes, and rejection of a single final success after an unstable probe
window. Full Go tests, agent race tests, `go vet`, script audits,
documentation consistency and the local self-test pass for the changed
surface. A live provider handshake remains deployment-specific operational
evidence and must be verified on the target node.
