# Release 7.1.1.17

**Release:** `7.1.1.17`

Release `7.1.1.17` makes external-provider policy-routing initialization
idempotent when a deployment routing table does not exist yet.

## Scope

- Treats `ipv4: FIB table does not exist` and equivalent missing-table output
  as an empty initial state while installing the external-egress fail-closed
  guard.
- Continues by creating the managed table with an `unreachable default` route
  before installing the deployment-specific fwmark rule.
- Keeps permission failures and every unknown route-flush error fatal.
- Aligns Apply behavior with the existing idempotent Cleanup behavior.

## Operator Action

Deploy `7.1.1.17` to the Control Plane and update the affected provider-node
agent to `7.1.1.17`. Retry deployment `Apply`; a separate Cleanup is not
required solely for an absent FIB table.

## Compatibility

- No PostgreSQL migration is required.
- API and job payload contracts are unchanged.
- Existing active external-egress deployments are unchanged.
- The fix applies to every external-egress protocol using managed policy
  routing.

## Validation

Unit tests verify successful guard initialization from a missing table, exact
command ordering, rejection of a route-flush permission failure and the
existing idempotent Cleanup behavior. Full Go tests, race tests, script audits,
static checks and release self-test are required before publication.
