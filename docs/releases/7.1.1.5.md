# Release 7.1.1.5

**Release:** `7.1.1.5`

Release `7.1.1.5` makes agent clock drift explicit and prevents SSH bootstrap
from reporting success when the resulting signed agent channel cannot operate.

## Scope

- Reports request-signature clock skew with direction, duration and NTP
  remediation in API logs and node diagnostics.
- Reports observed control-plane clock skew in agent logs when a signed response
  falls outside the replay-protection window.
- Runs a remote UTC preflight over the host-key-pinned SSH channel before
  issuing enrollment material or installing the agent.
- Blocks SSH bootstrap when node and control-plane worker clocks differ by more
  than two minutes while preserving the five-minute signed-channel window.
- Prevents invalid bearer tokens from changing the operator-visible
  authentication state of arbitrary nodes.
- Preserves enrollment identity and queued node jobs. Clock recovery does not
  require another re-enrollment.

## Operator Action

Synchronize NTP on both the control-plane host and the affected node, verify
`date -u +%s` is close on both systems, then restart `megavpn-agent`. Use
`systemd-timesyncd` or chrony according to the host configuration. Do not
disable signature enforcement or increase the signature window.

## Compatibility

- No PostgreSQL migration is required.
- Existing node identities, service instances and queued jobs are preserved.
- SSH bootstrap now fails early with `stage=clock_preflight` when clock drift is
  unsafe.
- Manual bootstrap bundles remain available, but the operator remains
  responsible for synchronizing the destination host before starting the
  agent.

## Validation

Regression coverage verifies clock-skew direction and diagnostics, fail-closed
SSH bootstrap preflight, valid-token-only node diagnostic updates, response
signature diagnostics, and the existing signed-channel behavior.
