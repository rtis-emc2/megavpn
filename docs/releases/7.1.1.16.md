# Release 7.1.1.16

**Release:** `7.1.1.16`

Release `7.1.1.16` hardens operational shell scripts and makes their safety
properties permanent CI and release-gate requirements.

## Scope

- Audits every shell entrypoint under `scripts/`, `deploy/` and the root
  deployment wrapper for syntax, supported shebang, executable mode and valid
  compatibility-wrapper targets.
- Audits Web UI and operational JavaScript syntax and validates root
  JavaScript compatibility-wrapper targets.
- Prevents `install-web.sh` from using filesystem root, protected system roots,
  source ancestors or source descendants as the target of `rsync --delete`.
- Prevents the Control Plane installer from accepting filesystem root or a
  protected system root as an application, Web UI or artifact directory.
- Rejects symbolic links, hardlinked regular files and special files in backup
  artifact and optional configuration trees before database dump or archive
  creation.
- Copies restore input into a private workspace and rejects absolute paths,
  parent traversal, links and special files in both outer and nested artifact
  archives before changing PostgreSQL.
- Extracts validated archives without restoring archived ownership or
  permissions.
- Corrects Web UI cache-key consistency checks for every four-component release
  version rather than only the historical `7.0.1.x` series.

## Operator Action

Deploy `7.1.1.16` to the Control Plane. No node-agent update is required for
runtime behavior, but keeping Control Plane and agent versions aligned remains
the supported operating model.

Before a production restore, verify that artifact data contains only regular
files and directories. A backup produced from a tree containing a symbolic
link, device, socket or FIFO now fails closed and must be investigated.

## Compatibility

- No PostgreSQL migration is required.
- Control Plane API and node-agent protocol contracts are unchanged.
- Existing valid backup archives remain compatible.
- Restore targets below normal application paths such as
  `/var/lib/megavpn/artifacts` remain supported.

## Validation

Regression smoke covers safe backup/restore, absolute and parent-traversal
archive paths, outer symlink and hardlink archives, nested symlink archives,
unsafe backup trees, protected restore targets and unsafe Web UI installation
targets. The release gate also runs the repository-wide shell and JavaScript
audits, Go tests, race tests, static checks and documentation consistency.
