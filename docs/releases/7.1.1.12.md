# Release 7.1.1.12

**Release:** `7.1.1.12`

Release `7.1.1.12` stabilizes the external provider gateway UI across desktop,
tablet and mobile viewports.

## Scope

- Fixes the shared modal helper so forms opened with `size: large` receive the
  wide layout requested by their page module.
- Reworks the external egress profile editor into stable profile and
  protocol-specific sections.
- Keeps the action bar visible while long provider configurations,
  certificates and private keys are edited.
- Preserves entered protocol and L2TP/IPsec authentication values when the
  operator switches between protocols or authentication modes.
- Improves provider-file import controls and clarifies which encrypted
  configuration is retained during profile editing.
- Converts the wide external egress deployment table into readable stacked
  rows on smaller screens instead of requiring horizontal scrolling.
- Adds a frontend regression assertion for large modal sizing.

## Operator Action

Deploy `7.1.1.12` to the Control Plane and refresh the browser once after the
deployment. Updated asset cache keys prevent stale CSS and JavaScript from
being reused.

No node-agent update is required for the UI changes in this release.

## Compatibility

- No PostgreSQL migration is required.
- Control Plane and node-agent API contracts are unchanged.
- Existing provider profiles, encrypted secrets and deployments are unchanged.

## Validation

The form was inspected at 1440, 1024, 768 and 390 pixel viewport widths,
including L2TP/IPsec PSK and certificate modes. The full self-test completed
with 20 passed and 0 failed checks; environment-specific PostgreSQL, systemd,
Nginx, live API and VPN data-plane gates remain deployment checks.
