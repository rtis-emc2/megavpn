# Release 7.1.1.1

**Release:** `7.1.1.1`

Release `7.1.1.1` is a stabilization and managed external-egress update for
the MegaVPN Control Plane, agent runtime and operator console.

## Scope

- Adds global external provider egress profiles with encrypted credentials and
  per-node deployments.
- Adds runtime-ready OpenVPN and WireGuard provider gateways. Other catalogued
  provider protocols remain explicitly planned and cannot be activated.
- Routes only selected VLESS client access groups through a provider gateway;
  the node main routing table and unrelated traffic are not changed.
- Uses a dedicated fwmark, routing table and fail-closed unreachable route for
  every managed provider deployment.
- Adds operator documentation for importing provider configurations,
  deployment, group assignment, validation, cleanup and failure recovery.

## Stabilization Fixes

- Automatic page refresh no longer replaces an open modal or a form while the
  operator is typing in an input, select, textarea or editable field.
- Client access-group preview and bulk dry-run are read-only and do not create
  instance revisions.
- A stale external-egress job result cannot overwrite a newer deployment job.
- Agent health distinguishes a real provider default route from the managed
  fail-closed unreachable route.
- WireGuard provider imports require exactly one peer for deterministic
  route ownership.
- External-egress cleanup verifies script execution, route-table cleanup and
  policy-rule removal before reporting success. Emergency node cleanup also
  removes managed external-egress rules and tables, including failed remnants.
- Profile deletion marks inactive deployment rows deleted and removes
  associated secret references during migration rollback.
- The client route form hides conflicting normal-egress controls when an
  external provider gateway is selected.
- External provider forms use protocol-aware fields, structured validation and
  consistent modal actions instead of browser alerts or confirmations.
- Imported OpenVPN and WireGuard profiles must pass preview before activation.
- Updates `golang.org/x/text` to `v0.39.0`, closing reachable
  `GO-2026-5970` reported by the release `govulncheck` gate through pgx.

## Security Properties

- Provider credentials and imported configurations are stored through the
  existing encrypted secret service and are not returned by list/read APIs.
- External runtime artifacts are written only to deployment-scoped managed
  paths with explicit mode and path allowlists.
- Provider traffic is selected by access-group policy. No automatic
  node-wide default-route replacement is performed.
- Missing provider routes fail closed for marked traffic instead of leaking it
  through the node default gateway.
- Job results are accepted only for the deployment's current job identity.

## Validation

The release is accepted through the maintained gates:

```bash
make build
go test ./...
go test -race ./...
go vet ./...
scripts/ci/docs-consistency.sh
scripts/ci/self-test.sh
scripts/ci/release-gate.sh
```

Live PostgreSQL migration/restore, systemd, nftables/iproute2 and provider data
plane checks remain environment-dependent release evidence and must be run on
a disposable node before production rollout.
