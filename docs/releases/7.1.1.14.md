# Release 7.1.1.14

**Release:** `7.1.1.14`

Release `7.1.1.14` makes managed L2TP/IPsec cleanup and replacement
ownership-aware and idempotent.

## Scope

- Preserves the managed XL2TPD PID evidence until process teardown completes.
- Verifies `/proc/<pid>/exe` before signaling a saved PID, preventing a reused
  PID or unrelated process from being terminated.
- Stops orphaned MegaVPN-managed `xl2tpd` processes with bounded TERM/KILL
  waits during both `Cleanup` and the next `Apply`.
- Recovers an orphan left by an older cleanup even when its PID file was
  already removed, but only when `ss`, the executable and the process command
  line prove MegaVPN ownership.
- Preserves managed artifacts when process ownership cannot be inspected, so a
  later cleanup can retry safely.
- Applies the same ownership-aware teardown to emergency cleanup.
- Verifies UDP/1701 after cleanup and before apply with process-owner details
  from `ss -lunp`.
- Reports a precise fail-closed error when an unrelated listener still owns
  UDP/1701.

## Operator Action

Deploy `7.1.1.14` to the Control Plane and update the node agent to
`7.1.1.14`. The agent update is required because the corrected L2TP lifecycle
is executed on the provider node.

For a deployment left inactive by the previous release, run `Cleanup` once and
then `Apply`. The updated agent removes a surviving managed XL2TPD process
before deleting its runtime state.

If UDP/1701 is still reported as occupied, inspect the owner without changing
the node:

```bash
sudo ss -H -lunp | grep ':1701 '
sudo systemctl status xl2tpd 'megavpn-external-egress-*' --no-pager -l
```

Do not stop the reported process until its ownership is confirmed.

## Compatibility

- No PostgreSQL migration is required.
- Control Plane API contracts are unchanged.
- Existing provider profiles, encrypted secrets and deployments are unchanged.
- Non-L2TP external provider gateways are unaffected.

## Validation

The regression suite covers managed orphan termination, reused-PID rejection,
cleanup ordering and UDP/1701 owner extraction. Go tests, agent race tests,
documentation consistency and the full release self-test pass for this release.
