# Release 8.0.0

Release `8.0.0` introduces the MegaVPN Console frontend architecture.

Current status: release candidate groundwork with VLESS client access groups,
Firewall preview/apply/disable, Clients core/artifacts, Clients delivery
workflows, existing Instances runtime control, service pack instance creation,
manual instance creation, service pack management and runtime artifact
list/import and Nodes observability/diagnostics/inventory workflows migrated to
the new console. Broader write-workflow parity is still tracked per domain.
Backend binary/version metadata remains `7.1.1.0` until the separate version
synchronization task is completed.

## Highlights

- New React + TypeScript + Vite frontend workspace under `frontend/`.
- New MegaVPN UI component kit and CSS design tokens.
- Russian and English i18n resources with key parity checking.
- Typed API client preserving cookie session and CSRF behavior.
- TanStack Query server-state layer.
- Legacy static UI preserved under `/legacy/`.
- Go static serving updated for `/legacy/`, Vite-built root assets and direct
  frontend deep links.
- CI/release-gate frontend checks expanded to typecheck, lint, test, i18n and build.
- Frontend serving smoke validates root UI, deep links, legacy UI, API
  non-shadowing and static asset 404 behavior.
- Frontend static guards block raw page-level API calls, auth token browser
  storage, unreviewed HTML sinks and production console logging.
- `Clients -> Groups -> VLESS` is wired to typed generic client access group
  endpoints for create/edit, member preview/apply/remove, scope update and sync
  preview/apply.
- `Clients` core workspace is wired for list/detail, create, activate/suspend,
  revoke/delete, single-client VLESS access assignment and client artifact
  list/build/download/delete.
- Client delivery workflows are wired for artifact share links, VLESS
  subscriptions and email delivery with transient one-time secret display.
- Existing `Instances` runtime control is wired for list/detail, runtime state,
  revisions/rollback, apply/reapply, lifecycle actions, diagnostics,
  delete/force-delete and job tracking.
- `Services -> Service Packs` is wired for list/detail, JSON create/update,
  enable/disable/delete and create instance from pack.
- `Instances` manual create and spec replace are wired to backend instance
  endpoints with explicit confirmation where required.
- `Services -> Runtime Artifacts` is wired for runtime binary artifact list,
  safe metadata view and URL import.
- `Nodes` is wired for existing node list/detail, agent/runtime state,
  maintenance mode, inventory view/sync, capability install/verify,
  diagnostics retry/run, service discovery list/import and async job tracking.

## Operator Workflow Impact

- Root `/` opens the new 8.0.0 console.
- `/legacy/` opens the previous static UI during migration.
- `Clients -> Groups` is the source of truth for access groups; the VLESS path
  is usable in the new console without `/legacy/`.
- `Clients` core operations are usable in the new console without `/legacy/`
  for client list/detail/create/status/delete/revoke, single-client VLESS
  preview/apply/remove and artifacts list/build/download/delete.
- `Clients -> Delivery` supports share link create/rotate/revoke, VLESS
  subscription create-or-rotate/revoke and email delivery in the new console
  without `/legacy/`.
- `Instances` runtime control works in the new console without `/legacy/` for
  existing service instances. Access group materialization is read-only and
  links operators to `Clients -> Groups`.
- `Services -> Service Packs` works in the new console without `/legacy/` for
  service pack list/detail/create/update/enable/disable/delete and create
  instance from pack.
- `Instances -> Manual instance` works in the new console without `/legacy/`
  where the backend accepts manual instance creation.
- `Instances -> Spec` replacement works in the new console without `/legacy/`
  through the backend spec replace endpoint and explicit confirmation.
- `Services -> Runtime Artifacts` works in the new console without `/legacy/`
  for list, metadata view and URL import.
- `Instances` no longer owns primary VLESS group management in the new information architecture.
- Firewall address groups, policies, rules, node preview, apply, node state and
  emergency disable are connected in the new UI without `/legacy/`.
- Firewall UI keeps preview-before-apply semantics.
- `Nodes` observability, diagnostics, inventory, capabilities and service
  discovery workflows for existing nodes are usable in the new console without
  `/legacy/`.
- Unsupported write workflows are disabled or linked to `/legacy/`; no
  fake-success actions are allowed.

## Known Limitations

- The first architecture pass implements the new shell, typed data loading and read-path pages.
- High-risk write workflows remain disabled or legacy-only unless the backend
  action is explicitly wired.
- Client access group preview/apply is fully connected for VLESS only.
  Non-VLESS access services remain catalog-only until backend materialization
  is implemented.
- Rule reorder remains unavailable because the backend has no reorder endpoint.
- Live/staging operator validation is still required before production use of
  strict firewall apply.
- Nodes bootstrap, host-key scan, SSH terminal/session launch, enrollment
  tokens, agent token rotation, reboot, emergency cleanup, retire/force-retire
  and route policy preview/apply/cleanup remain FE8-P0-05B or legacy-only.
- Node runtime reconcile is connected through the backend diagnostics endpoint;
  it may queue dependent runtime jobs as defined by backend logic and still
  requires live/staging operator validation before production reliance.
- Runtime artifact delete remains unavailable because the backend has no binary
  runtime artifact delete endpoint in this release.
- Service pack create/update has no separate validation endpoint; backend
  validation runs during the real create/update mutation.
- Instance spec replace has no separate preview endpoint or draft-save HTTP
  route in this release.
- Client core provisioning through single-client VLESS access assignment,
  client artifacts, share links, VLESS subscriptions and email delivery are
  connected in the new console. Client routes, access rotation and config
  cleanup remain legacy-only.
- Client delivery history remains unavailable because the backend has no
  client-scoped delivery history list endpoint in this release.
- Client email delivery is real and synchronous, but the backend sends the
  client's available artifacts/configs and does not accept an artifact-specific
  email payload yet.
- Certificates write workflows and Platform settings save/apply/test actions
  remain legacy-only.
- Backhaul create/apply/probe/promote/route/delete mutations remain legacy-only.
- Full workflow parity should be completed page by page against
  `docs/FRONTEND_AUDIT_8.0.0.md`,
  `docs/FRONTEND_ENDPOINT_PARITY_8.0.0.md` and
  `docs/FRONTEND_WRITE_WORKFLOWS_8.0.0.md`.
- Backend binary version synchronization to `8.0.0` is still required before
  final release readiness.
- Final 8.0.0 cutover remains NO-GO until the remaining legacy-only workflows,
  live/staging operator validation and version synchronization are complete.
