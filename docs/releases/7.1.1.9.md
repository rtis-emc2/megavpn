# Release 7.1.1.9

**Release:** `7.1.1.9`

Release `7.1.1.9` completes the external provider gateway deployment lifecycle
after cleanup.

## Scope

- Allows an inactive external egress deployment to be reactivated with `Apply`.
- Restores `desired_status=active` transactionally when the reactivation job is
  queued.
- Adds removal of an inactive node deployment without deleting the provider
  profile or its encrypted credentials.
- Requires successful cleanup and no active deployment jobs before removal.
- Keeps PostgreSQL L2TP/IPsec exclusivity guards active during reactivation.
- Replaces unusable inactive-state actions with explicit `Reactivate` and
  `Remove deployment` controls.
- Adds API and PostgreSQL lifecycle regression coverage.

## Operator Action

Deploy `7.1.1.9` to the control plane. The node agent from `7.1.1.8` remains
compatible with this control-plane-only lifecycle fix.

For an inactive deployment:

1. Select `Reactivate` to install/materialize the provider runtime again.
2. Or select `Remove deployment` to remove only the profile-to-node assignment.

The provider profile remains available for deployment on another node.

## Compatibility

- No PostgreSQL migration is required.
- Existing profiles, encrypted secrets, group assignments and deployment
  resource identities are preserved.
- Removed deployments are soft-deleted for auditability and can be recreated
  on the same profile/node without allocating duplicate resources.

## Validation

The full Go test suite, API handler tests, PostgreSQL lifecycle integration
coverage, JavaScript syntax validation, documentation consistency and release
self-tests form the promotion evidence.
