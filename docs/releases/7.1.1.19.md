# Release 7.1.1.19

**Release:** `7.1.1.19`

Release `7.1.1.19` repairs the generated L2TP/xl2tpd client runtime, makes
failed provider handshakes diagnosable from job evidence and restores the
Control Plane dependencies required by the browser SSH terminal.

## Scope

- Replaces the invalid xl2tpd `max redials = 0` setting with a bounded positive
  retry count accepted by current xl2tpd packages.
- Rate-limits failed external-egress systemd starts and increases the L2TP
  restart delay, preventing an invalid provider runtime from entering a tight
  restart loop.
- Captures bounded, redacted systemd, xl2tpd, pppd and strongSwan evidence for
  failed L2TP Apply and Probe jobs.
- Classifies common provider failures, including IKE timeout, proposal
  mismatch, IPsec authentication failure, PPP authentication failure,
  UDP/1701 conflict and rejected generated configuration.
- Installs and verifies `openssh-client` and `sshpass` on Control Plane hosts
  deployed by supported scripts so the browser SSH terminal has its required
  runtime commands.
- Uses long-lived WebSocket proxy timeouts in the generated Control Plane Nginx
  configuration and preserves actionable terminal errors in the UI.

## Operator Action

Deploy `7.1.1.19` to the Control Plane and update the affected node agent to
`7.1.1.19`. Re-run Apply for the L2TP deployment so the agent replaces the old
generated xl2tpd configuration and systemd unit.

If the new job evidence reports that the provider IKE endpoint did not respond,
verify bidirectional UDP/500 and UDP/4500 reachability, upstream NAT, provider
source-IP allowlists and the configured provider endpoint. That condition is
outside the generated xl2tpd configuration and must be resolved before an
IPsec security association can form.

For an existing Control Plane installed without the standard installer,
install `openssh-client` and `sshpass`, preserve WebSocket
`Upgrade`/`Connection` headers in Nginx and use read/send timeouts suitable for
interactive terminal sessions.

## Compatibility

- No PostgreSQL migration is required.
- API and external-egress job payload contracts are unchanged.
- Existing provider profiles and encrypted credentials are unchanged.
- A failed L2TP deployment must be applied again after the node agent update to
  regenerate its managed files.

## Validation

Tests cover valid xl2tpd retry rendering, systemd restart limiting, L2TP failure
classification and diagnostic secret redaction. Full Go tests, the changed
agent and HTTP race suites, JavaScript syntax checks, shell syntax checks and
documentation consistency pass for the release surface. The provider-side IKE,
PPP and Internet data plane remain live deployment evidence.
