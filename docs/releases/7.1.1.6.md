# Release 7.1.1.6

**Release:** `7.1.1.6`

Release `7.1.1.6` fixes stale agent-channel degradation after a node recovers
from an authorization or signed-request failure.

## Scope

- Uses PostgreSQL transaction time for both node heartbeat and agent last-seen
  timestamps.
- Clears the previous agent authorization failure only after a successfully
  authenticated and signature-verified heartbeat.
- Updates node state, agent state and recovery diagnostics in one transaction.
- Adds regression coverage for authorization-failure recovery while preserving
  agent version projection.

## Operator Action

Deploy the control-plane API and worker, then update node agents to `7.1.1.6`.
No re-enrollment or database cleanup is required. A healthy agent clears stale
authorization diagnostics with its next heartbeat.

## Compatibility

- No PostgreSQL migration is required.
- Existing node identities, enrollment tokens, jobs and service instances are
  preserved.
- The five-minute signed-channel timestamp window remains unchanged.
- Invalid tokens still cannot change node diagnostics.

## Validation

The full Go test suite passes. PostgreSQL integration coverage verifies that a
successful heartbeat removes an earlier authorization failure marker.
