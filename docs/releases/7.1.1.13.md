# Release 7.1.1.13

**Release:** `7.1.1.13`

Release `7.1.1.13` separates reusable external-provider profiles from their
node-specific runtime deployments.

## Scope

- Replaces the combined profile/deployment table with `Profiles` and
  `Deployments` tabs.
- Keeps provider credentials, endpoints and profile lifecycle actions in the
  profile view.
- Moves node runtime status, diagnostics and apply/probe/cleanup actions into a
  dedicated deployment view.
- Gives deployment errors a full readable column instead of compressing them
  between status and action controls.
- Uses a clear action hierarchy: `Apply` is primary, with `Probe` and
  `Cleanup` below it.
- Converts both tables into stable stacked records on tablet and mobile
  viewports.
- Adds frontend regression coverage for the two-tab responsibility boundary.

## Operator Action

Deploy `7.1.1.13` to the Control Plane and refresh the browser once after
deployment. Updated asset cache keys prevent stale CSS and JavaScript from
being reused.

No node-agent update is required for these UI changes.

## Compatibility

- No PostgreSQL migration is required.
- Control Plane and node-agent API contracts are unchanged.
- Existing profiles, encrypted secrets and node deployments are unchanged.
- Existing apply, probe, cleanup, reactivation and removal behavior is
  preserved.

## Validation

The page was inspected at 1440, 1024 and 390 pixel viewport widths. Both tabs
render without document-level horizontal overflow, clipped status text or
overlapping actions. The full release self-test completed with 20 passed and
0 failed checks; environment-specific PostgreSQL, systemd, Nginx, live API and
VPN data-plane gates remain deployment checks.
