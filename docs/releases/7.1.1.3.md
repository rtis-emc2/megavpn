# Release 7.1.1.3

**Release:** `7.1.1.3`

Release `7.1.1.3` completes the operator-facing external provider gateway
workflow introduced in `7.1.1.2`. The release does not change database schema
or node-wide routing semantics.

## Scope

- Makes external egress profile keys system-generated and immutable.
- Exposes only protocols with a runtime-ready driver in the operator catalog.
- Replaces the generic profile editor with protocol-specific forms.
- Adds an explicit L2TP over IPsec workflow for provider endpoint, PPP
  credentials and PSK or client-certificate authentication.
- Accepts a DNS name, IP address, IPv4 `/32` or IPv6 `/128` as the L2TP
  provider endpoint. Network CIDRs containing multiple hosts are rejected.
- Validates certificate chains, certificate validity and private-key matching
  before storage and again on the target node.
- Removes obsolete PSK or certificate secrets when the selected authentication
  mode changes.
- Preserves an existing encrypted provider configuration when an operator edits
  only profile metadata or rotates credentials.

## Runtime And Security Model

External provider routing remains scoped to explicitly assigned client access
groups. The node main routing table is not replaced and unselected client or
node traffic keeps its existing path.

L2TP certificate material is stored in encrypted secret references. During
apply, the agent writes deployment-specific files with mode `0600` under the
strongSwan certificate directories. Cleanup removes only the exact managed
paths. Certificate mode does not retain an obsolete PSK; PSK mode does not
retain obsolete certificate material.

## Compatibility

- Existing external egress profiles retain their internal profile key.
- Existing L2TP/IPsec PSK profiles remain compatible.
- Planned and preview protocol definitions remain in the internal roadmap
  catalog but are no longer exposed as selectable operator options.
- No PostgreSQL migration is required for this release.

## Validation

The release was validated with:

```bash
go test ./...
go test -race ./...
scripts/ci/docs-consistency.sh
scripts/ci/self-test.sh
scripts/ci/release-gate.sh
```

The self-test may skip disposable PostgreSQL, backup/restore, systemd, Nginx,
live API and VPN service-matrix checks when their explicit release environment
variables are unavailable. L2TP/IPsec negotiation with real provider
certificates still requires disposable Ubuntu node evidence before production
promotion.
