# Release 7.1.1.10

**Release:** `7.1.1.10`

Release `7.1.1.10` fixes automatic Ubuntu package recovery for L2TP/IPsec
external provider gateways.

## Scope

- Continues package recovery after an initial `dpkg --configure -a` dependency
  failure instead of terminating the apply job prematurely.
- Runs `apt-get -f install`, repeats `dpkg --configure -a`, and only then retries
  the requested runtime package installation.
- Installs and verifies L2TP/IPsec runtime dependencies in the explicit order
  strongSwan, PPP and XL2TPD.
- Adds the missing `pppd` runtime verifier so a successful PPP install is not
  reported as an unsupported capability.
- Preserves structured command, exit-code and output evidence for every repair
  step.
- Documents the manual recovery sequence for nodes that already contain an
  interrupted apt/dpkg transaction.
- Adds regression coverage for the observed XL2TPD dependency failure.

## Operator Action

Deploy `7.1.1.10` to the Control Plane and update the affected node agent. If
the node already has an interrupted package transaction, run the documented
repair sequence in `docs/EXTERNAL_EGRESS.md`, verify `ipsec`, `pppd` and
`xl2tpd`, then run deployment `Apply` again.

The external egress profile, encrypted credentials, access-group assignments
and deterministic deployment resources do not need to be recreated.

## Compatibility

- No PostgreSQL migration is required.
- Agent and Control Plane API contracts are unchanged.
- Existing active external egress deployments are not modified until an
  operator applies them.

## Validation

Targeted package-repair regression tests, the full Go test suite, race tests,
vet/build checks, JavaScript syntax validation, documentation consistency and
release self-tests form the promotion evidence.
