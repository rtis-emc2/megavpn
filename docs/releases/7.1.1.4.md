# Release 7.1.1.4

**Release:** `7.1.1.4`

Release `7.1.1.4` restores deterministic agent-channel diagnostics after a
control-plane deployment and prevents stale heartbeat state from being shown
as an online agent.

## Scope

- Signs agent API error responses, including authorization and validation
  failures, with the token presented by the agent.
- Keeps fail-closed response-signature verification on the node.
- Reports HTTP status, content type and a bounded response preview when a proxy
  or incompatible endpoint returns an unsigned response.
- Derives the operator-visible agent channel state from heartbeat freshness
  instead of the persisted last-known `online` value.
- Preserves the existing external-egress job claim path and queued jobs. Once
  the node is securely re-enrolled, pending node jobs are claimed normally.

## Operator Action

An agent that reports `agent unauthorized` has a local identity that no longer
matches the active server-side identity. Use the node **Re-enroll agent**
workflow over the configured SSH bootstrap channel. Do not disable signature
enforcement and do not copy agent tokens manually.

## Compatibility

- No PostgreSQL migration is required.
- Existing queued jobs and external-egress profiles are preserved.
- Agents on `7.1.1.3` can verify the newly signed error response.
- Re-enrollment installs the `7.1.1.4` agent and keeps the existing node ID.

## Validation

The release is covered by Go regression tests for signed agent errors and
unsigned proxy diagnostics, frontend smoke coverage for stale heartbeat state,
and the standard self-test/release gates.
