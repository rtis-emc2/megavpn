# Release 7.1.1.7

**Release:** `7.1.1.7`

Release `7.1.1.7` fixes the heartbeat persistence regression introduced in
`7.1.1.6`.

## Scope

- Clears the non-null authorization failure reason with an empty string instead
  of SQL `NULL`.
- Preserves transactional heartbeat and agent recovery updates.
- Returns `404` only when the node lifecycle row is actually absent or retired.
- Returns a signed `500` and writes an API error log for database persistence
  failures instead of reporting the node as unregistered.
- Adds API regression coverage for missing-node and database-failure responses.

## Operator Action

Deploy `7.1.1.7` to the control plane immediately. Existing `7.1.1.6` agents
are protocol-compatible and reconnect automatically after the API restart. No
re-enrollment or direct database modification is required.

## Compatibility

- No PostgreSQL migration is required.
- Node identities, agent tokens, jobs and service instances are preserved.
- Signed request verification and replay protection remain unchanged.

## Validation

Targeted API and PostgreSQL tests and the full Go test suite pass. The
PostgreSQL integration test now matches the production `NOT NULL` schema
invariant for `last_auth_failure_reason`.
