# Release 7.1.1.8

**Release:** `7.1.1.8`

Release `7.1.1.8` makes external provider gateway cleanup safe after a partial
or failed runtime apply.

## Scope

- Treats an absent managed policy-routing table as an already-clean state.
- Keeps permission, malformed command and residual route/rule failures fatal.
- Skips the StrongSwan reload during L2TP/IPsec cleanup when `ipsec` was never
  installed.
- Resolves and invokes the installed `ipsec` executable by its discovered path.
- Requires the actual `ipsec` executable during capability verification instead
  of accepting an unrelated StrongSwan helper binary.
- Applies the same absent-table semantics to emergency node cleanup.
- Adds regression tests for partial external egress cleanup.

## Operator Action

Deploy `7.1.1.8` to the control plane and update the affected node agent to
`7.1.1.8`. Then run `Cleanup` again for the failed external egress deployment.
After cleanup succeeds, run `Apply`; the agent will install and verify
StrongSwan, xl2tpd and PPP before materializing the L2TP/IPsec runtime.

The previously failed cleanup job is retained as audit evidence and is not
replayed automatically.

## Compatibility

- No PostgreSQL migration is required.
- External egress profiles, encrypted secrets, deployments and client access
  group assignments are preserved.
- No routing table is created solely to clean an incomplete deployment.
- No package is installed solely to clean an incomplete deployment.

## Validation

Agent regression tests and the full Go test suite pass. Documentation
consistency and release self-tests are part of the promotion gate.
