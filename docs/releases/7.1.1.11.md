# Release 7.1.1.11

**Release:** `7.1.1.11`

Release `7.1.1.11` fixes package-manager isolation and failure diagnostics for
L2TP/IPsec external provider gateways.

## Scope

- Delegates only fixed `apt-get` and `dpkg` install/repair commands to a
  bounded transient systemd unit when the agent itself runs under systemd.
- Keeps the long-running agent unit hardened while allowing trusted Ubuntu
  package maintainer scripts to configure PPP, XL2TPD and strongSwan.
- Collects the transient helper after completion and records its unit name,
  exit code and output in job evidence.
- Reports the last meaningful `apt`/`dpkg` diagnostics instead of the first
  progress line such as `Reading package lists...`.
- Resolves `pppd` from standard absolute system paths during runtime
  verification.
- Makes L2TP apply replace stale managed XL2TP runtime and stop the conflicting
  distribution XL2TPD service before claiming UDP/1701.
- Removes managed L2TP units, routes, runtime state, IPsec include directives
  and certificate material during deployment and emergency cleanup.
- Purges XL2TPD, PPP and strongSwan executable packages during confirmed full
  node wipe; a purge failure keeps the agent installed for recovery.
- Adds regression coverage for package command classification, transient-unit
  wrapping, diagnostic summarization and L2TP cleanup boundaries.
- Fails full-node wipe when package state cannot be verified or unmanaged
  L2TP/IPsec executables remain.
- Bounds unauthenticated rate-limit and agent-signature replay caches to prevent
  unbounded memory growth.
- Rejects static web asset symlink escapes after resolving the real filesystem
  path.
- Rejects malformed operator invite resend payloads instead of silently using
  default values.
- Propagates VLESS client-access-group metadata synchronization failures instead
  of reporting a partially materialized instance as successful.
- Documents the complete security audit, closed findings and remaining
  deployment constraints in `docs/SECURITY_REVIEW_7.1.1.11.md`.

## Operator Action

Deploy `7.1.1.11` to the Control Plane, update the affected node agent and run
deployment `Apply` again. If the package database is already interrupted, use
the recovery sequence in `docs/EXTERNAL_EGRESS.md` first.

The external egress profile, encrypted credentials, group assignments and
deployment record do not need to be recreated.

## Compatibility

- No PostgreSQL migration is required.
- Agent and Control Plane API contracts are unchanged.
- Existing active external egress deployments are not modified until an
  operator applies them.

## Validation

Targeted API, PostgreSQL and agent tests, the full Go suite, race tests,
vet/build checks, JavaScript and shell syntax validation, documentation
consistency, static security checks and the release self-test form the
promotion evidence. Disposable PostgreSQL migration/restore and live
Linux/VPN data-plane checks remain environment-specific promotion gates.
